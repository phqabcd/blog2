IoC（控制反转）与DI（依赖注入）

Spring中IoC容器的实现原理

介绍：IoC（控制反转）思想是将对象的创建和管理，交给外部环境。
IoC容器、DI（依赖注入）是Sprign框架中的实现方式；
IoC容器实际上是个存放各种对象的Map。

Bean的生命周期与作用域

介绍：Bean代指的是被IoC容器管理的对象，这些对象通过配置元数据来定义，配置元数据可以是XML文件、注解或者Java配置类。

生命周期

  1. 创建
    找到配置文件中的 Bean 定义，然后使用 Java 反射 API 来创建 Bean 的实例。
  2. 属性赋值/填充
    为 Bean 设置相关属性和依赖，例如@Autowired 等注解注入的对象、@Value 注入的值、setter方法或构造函数注入依赖和值、@Resource注入的各种资源。
  3. 初始化
    Aware 接口的依赖注入、BeanPostProcessor 在初始化前后的处理以及 InitializingBean 和 init-method 的初始化操作。
  4. 销毁
    注册相关销毁回调接口，最后通过DisposableBean 和 destory-method 进行销毁。

作用域

  • singleton：默认作用域，在IOC容器中只有唯一的bean实例；
  • protorype：每次获取都会创建一个新的bean实例；
  • 其他：request（每个HTTP请求产生一个新的bean）；session（每个session产生一个新的bean）；application/global-session （每个 Web 产生一个新的bean）；websocket （每个 WebSocket 会话产生一个新的 bean）。

AOP（面向切面编程）

介绍：将那些与业务无关，为业务模块所共同调用的逻辑（如事务处理、日志管理、权限控制等）封装起来。

Spring AOP的实现机制（基于动态代理）

如果要代理的对象实现了某个接口，通过JDK Proxy创建代理对象；
对于没有实现接口的对象，使用Cglib生成一个被代理对象的子类来作为代理；

切面、通知、连接点等概念解析

  • 切面：切入点 + 通知；
  • 切入点：被切面拦截/增强的连接点；
  • 连接点：目标对象所属类中，定义的方法；
  • 通知：增强的逻辑/代码，拦截到目标对象的连接点后要做的事情，包括前置、后置、返回、异常、环绕通知；
  • 目标：被通知的对象；
  • 代理： 将通知应用到目标对象，生成的代理对象；
  • 织入：将通知应用到目标对象，生成代理对象的动作过程；
